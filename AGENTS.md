# Repository Instructions

- Keep `.env` and all generated data/model artifacts out of git.
- Run lints, type checks, and targeted tests before committing.
- Commit finished work to a branch. Push and open a PR when a remote is
  configured.
- Treat human behavioral claims and neural claims carefully: Prolific,
  in-person, fMRI, EEG, fNIRS, or hyperscanning validation is required before
  making claims about real human effects.
